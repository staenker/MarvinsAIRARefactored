Racing Wheel — Device

Controls that select your wheelbase and manage basic force‑feedback functions.

Racing Wheel Device FFB Basics
UI Location
Racing Wheel page • section labeled DEVICE
What this section does
Lets you select which wheel MAIRA controls and provides quick actions to enable/disable output, play a test pulse, and re‑initialize the connection.

Steering Device

Dropdown labeled Steering Device.

  • Default: Not selected (empty GUID).
  • What it does: selects the DirectInput force‑feedback device MAIRA will control.
  • How it works: MAIRA lists detected FFB‑capable devices. When you pick one, MAIRA saves the device’s GUID and binds output to it. If the device is offline at launch, the previous selection is preserved and flagged as “Device not found”.

Power

Button labeled Power.

  • Default: On.
  • What it does: When On, MAIRA sends FFB to the selected device; when Off, MAIRA does not output FFB.
  • Details: Supports button mappings and optional context switches (per wheelbase / car / track / layout / wet‑dry).
Pro tip: map an easily reachable button to Power so you can instantly cut FFB if the wheel behaves unexpectedly (e.g., oscillation or runaway force). This gives you a way to perform a quick emergency shut‑off.
Red
Force-feedback has been manually shut off.
Orange
On, but force-feedback has been automatically disabled due to iRacing simulator not running, or initialization error.
Yellow
On, but force-feedback has been automatically disabled due to iRacing FFB being enabled.
Green
On, and force-feedback is enabled and running normally.

Test

Button labeled Test.

  • What it does: Plays a brief (~2s) vibration pulse to confirm FFB is flowing.
  • Notes: Requires Power to be On and green.

Reset

Button labeled Reset.

  • What it does: Re‑initializes the FFB session with the selected device; useful after hot‑plugging or if output feels stuck.
Pro tip: map a button to Reset so you can quickly reset the connection if the wheel becomes disconnected or uninitialized for some reason.
